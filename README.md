# 电商公司运营数据可视化大屏设计方案

## 一、设计目标与使用场景
本方案用于电商管理层每日经营复盘与异常监控，覆盖淘宝、京东、拼多多、天猫四平台运营数据，并联动客服、仓储、推广、售后、运营项目与美工进度。

- 使用频率：每日例会/班后复盘（T+0或T+1日数据）
- 刷新机制：数据源每日更新一次，大屏支持手动刷新与定时重载页面
- 核心价值：快速识别“销售波动、退款异常、客服风险、库存风险、项目延期”

---

## 二、整体视觉与布局规范（可直接给前端）

### 1）画布与栅格
- 分辨率：1920×1080（16:9），兼容 1366×768 按比例缩放
- 栅格：24列栅格，左右边距 24px，模块间距 12px
- 顶部栏高度：72px
- 主体区分三列：
  - 左列 6/24：销售结构 + 客服
  - 中列 12/24：核心KPI + 趋势 + 项目甘特
  - 右列 6/24：仓储 + 推广 + 售后 + 美工

### 2）主题风格（深色高对比）
- 背景：#0B1220（主背景）、#111A2E（卡片底）
- 主文字：#E6EDF7；次级文字：#9FB0C7
- 主强调色：#2F7BFF（销售）、#2ED573（健康）、#FFB020（预警）、#FF4D4F（异常）
- 边框/分隔：#23314D
- 字体建议：`PingFang SC` / `Microsoft YaHei`，数字使用等宽字体（如 `DIN`）

### 3）信息层级
- 一级：当日总销售额、订单量、退款率、客服满意度、发货异常率
- 二级：平台拆分、推广效率、售后处理效率、库存告警
- 三级：SOP待办、项目任务、美工任务明细


### 4）左侧菜单栏（导航与管理）
- 位置：页面最左侧固定导航栏（建议宽度 220px，可折叠为 64px 图标栏）
- 菜单结构：支持一级/二级菜单，默认按业务流程排序（销售→客服→仓储→推广→售后→项目→美工）
- 管理操作：
  - 新增：`新增菜单` 按钮（弹窗录入菜单名称、编码、排序值、可见角色）
  - 编辑：每项菜单提供 `编辑` 按钮（修改名称、关联模块、排序规则）
  - 删除：每项菜单提供 `删除` 按钮（需二次确认）
- 菜单名称策略（前后端解耦）：
  - 前端展示名称 `menu_display_name` 支持自由修改并保存
  - 后端关联键 `menu_key` / `menu_id` 固定不变，用于接口权限与模块路由
  - 保存时仅更新展示名称，不改变后端映射关系，避免历史权限与统计口径错乱
- 自动排序：
  - 支持按 `sort_order` 自动升序展示
  - 支持“上移/下移”快捷调整；保存后实时更新菜单顺序
  - 若未配置排序值，则按创建时间升序兜底
- 审计要求：记录菜单配置变更日志（操作人、时间、变更前后值）


### 5）用户管理与菜单权限控制
- 新增“用户管理”界面，用于配置每位员工可浏览的菜单权限。
- 权限配置方式：
  - 左侧用户列表（支持按部门/角色筛选）
  - 右侧菜单树勾选框（父子级联勾选，可半选）
  - 操作按钮：`全选`、`清空`、`保存`
- 保存规则：
  - 点击`保存`后写入 `user_menu_permission` 关系表
  - 保存成功后提示“权限已更新”，并记录操作日志（操作人/时间/目标用户）
- 生效策略：
  - 实时生效（用户下次刷新页面后按新权限展示）
  - 无权限菜单在前端隐藏，接口侧同时做权限校验

### 6）菜单内表头管理与撤销机制
- 每个菜单页面支持自定义表头（列）管理：
  - `添加表头`：从可选字段池勾选新增列
  - `删除表头`：移除当前页面列（需二次确认）
- 表头配置字段建议：`column_key`、`column_name`、`visible`、`order`、`width`
- 提供 `恢复上一步` 按钮：
  - 使用前端操作栈缓存最近 2 步（例如“添加表头”“删除表头”）
  - 连续点击可最多回退 2 次，超过范围按钮置灰
  - 一旦点击“保存配置”，撤销栈清空并以最新配置为基线

## 三、页面布局草图（文字版）

### 顶部全局栏（72px）
- 左：系统标题「电商运营管理大屏」
- 中：日期选择器（默认当天，支持历史日期）+ “昨日/今日快捷切换”
- 右：
  - 数据更新时间（示例：2026-04-25 08:10）
  - 刷新按钮（手动）
  - 自动刷新开关（默认每10分钟重载页面，仅拉取同日数据快照，不改变“每日更新一次”数据节奏）

### 第一行（核心经营总览）
- 4~6个KPI卡片（横向）：
  1. 当日总销售额
  2. 当日订单量
  3. 客单价
  4. 退款率
  5. 客服满意度
  6. 发货异常率
- 每张卡片包含：当日值、较前一日环比、异常箭头（红升绿降或反向按业务定义）

### 第二行（左中右主业务区）
- 左列：
  - 模块A：多平台销售占比（饼图/环图）
  - 模块B：平台卡片轮播（淘宝、京东、拼多多、天猫）
- 中列：
  - 模块C：销售趋势（近7/30天折线）+ 平台筛选
  - 模块D：推广费用占销售额比（仪表+趋势）
- 右列：
  - 模块E：仓储发货与库存告警
  - 模块F：售后退款概览

### 第三行（执行层）
- 左列：客服绩效与SOP待办
- 中列：运营项目甘特图
- 右列：美工任务进度列表

## 四、各模块图表与字段设计

## 1）多平台销售模块

### 1.1 顶部总览卡片
- 指标：`total_sales_amount`、`order_count`、`avg_order_value`
- 图表：指标卡（带环比mini sparkline）
- 交互：点击卡片可联动下方趋势图

### 1.2 平台销售占比
- 图表建议：环形图（主）+ 柱状图（切换）
- 维度：`platform`（淘宝/京东/拼多多/天猫）
- 指标：`sales_amount`、`order_count`
- 交互：
  - 点击图例过滤平台
  - Hover显示销售额、占比、订单量

### 1.3 平台独立缩略卡
每个平台一张卡：
- 当日销售额
- 当日订单量
- 客单价
- 环比（较前一天）
- 退款率
支持自动轮播与手动左右切换。

---

## 2）客服模块

### 2.1 客服核心指标
- 指标：
  - `service_ticket_count`（接待总量）
  - `avg_first_response_sec`（平均首次响应时长）
  - `csat_score`（满意度评分，5分制）
- 图表：3张KPI卡 + 响应时长趋势线

### 2.2 SOP待办列表
- 字段：`task_type`（催单处理/退货审核/投诉跟进）、`owner`、`deadline`、`priority`、`status`
- 展示：表格/任务清单
- 颜色：
  - 急迫：红色标签（`priority=high` 或 `deadline` 当天）
  - 正常：蓝/灰标签

---

## 3）仓储模块

### 3.1 发货履约
- 指标：`shipment_count`、`pending_shipment_count`、`shipment_exception_rate`
- 图表：
  - 发货/待发货：并列柱状
  - 异常率：仪表盘或子弹图

### 3.2 库存预警
- 维度：SKU（商品）
- 字段：`sku_name`、`stock_qty`、`safety_stock`、`available_days`
- 规则：`stock_qty < safety_stock` 标红高亮，按缺口从大到小排序
- 产品预设范围（用于SKU字典）：
  连续封口机、卷膜、双室真空机、封箱机、捆扎机、封包机、封罐机、真空机、膏体机、茶叶烘焙机、袋子、压缩打包机、外抽真空机、保鲜膜打包机+膜、米砖模具、大米整形机、热收缩膜机、分装机（上体）。

---

## 4）推广与营销模块

### 4.1 推广宝贝表现
- 图表：横向条形图 + 明细表
- 字段：`campaign_item_name`、`spend`、`impressions`、`ctr`、`cvr`、`roi`
- 排序：默认按 `spend` 降序，可切换 `roi`

### 4.2 推广费占比
- 指标：`promotion_spend_ratio = spend / total_sales_amount`
- 展示：
  - 主数值卡（百分比）
  - 7日/30日趋势折线
- 阈值：
  - >15% 黄色预警
  - >25% 红色预警（阈值可配置）

---

## 5）售后与退款模块

### 5.1 当日退款概览
- 指标：`refund_apply_count`、`refund_amount`
- 图表：双指标卡 + 环形分类图
- 分类维度：`refund_reason`（质量问题、物流延迟、七天无理由、其他）

### 5.2 售后处理效率
- 指标：`after_sales_completion_rate`、`avg_after_sales_handle_hours`
- 图表：
  - 完成率：进度条/仪表盘
  - 处理时长：柱线组合（按日）

---



## 7）每日销售额（个人日销）模块

### 7.1 菜单入口与定位
- 左侧菜单新增：`每日销售额（个人日销）`。
- 用途：用于查看并提交“按人按店铺”的日销与询单转化数据。

### 7.2 提交字段（参考询单转化格式）
- 维度字段：`dt`、`store_name`、`staff_name`
- 指标字段：
  - `reception_count`（接待人数）
  - `after_sales_count`（售后人数）
  - `invalid_inquiry_count`（无效询单）
  - `pre_sales_count`（售前人数）
  - `deal_count`（成交人数）
  - `daily_sales_amount`（日销售额）
- 校验规则：
  - 各人数字段必须为非负整数
  - `deal_count <= pre_sales_count`
  - `daily_sales_amount >= 0`

### 7.3 提交与回显
- 提交方式：
  - 提供“保存草稿”“正式提交”两个按钮
  - 正式提交后进入“今日提交情况”统计
- 提交后展示：
  1. 今日提交情况：已提交人数、未提交人数、提交率、最后提交时间
  2. 数据明细：按店铺/人员展示上述字段，支持排序与导出
- 异常提示：
  - 若 `invalid_inquiry_count / reception_count > 30%` 标黄
  - 若 `deal_count / pre_sales_count < 10%` 标红

---

## 6）运营项目与美工进度模块

### 6.1 运营项目甘特图
- 字段：`project_name`、`owner`、`plan_end_date`、`actual_progress`、`status`
- 图形：甘特条 + 今日线（Today line）
- 规则：
  - `actual_progress < expected_progress` 且临近截止 -> 橙/红色
  - 已完成 -> 绿色

### 6.2 美工任务列表
- 任务类型：主图设计、详情页制作、活动海报
- 字段：`design_task_name`、`assignee`、`status`、`eta`
- 状态色：待开始（灰）、进行中（蓝）、已完成（绿）、逾期（红）

---

## 五、交互逻辑与异常标注规则

### 1）日期筛选与历史查询
- 支持选择任意历史日期 `dt`
- 所有模块以 `dt` 作为统一过滤条件
- 提供“前一天/后一天”快捷按钮

### 2）刷新与更新策略（关键约束）
- 数据仓库每日批处理一次（例如每日08:00完成）
- 大屏默认展示“最近已更新批次”
- 页面自动刷新仅用于拉取同批次修正值，不改变日更事实

### 3）悬停与下钻
- 所有图表 Hover 显示详细数值、占比、环比
- 点击平台/类目/任务可联动其他模块过滤
- 支持“重置筛选”一键恢复

### 4）关键指标异常规则（可配置）
- 销售额波动：`abs((today - yesterday)/yesterday) > 10%`
- 退款率异常：`refund_rate > 5%`
- 客服满意度风险：`csat_score < 4.0`
- 仓储异常：`shipment_exception_rate > 2%`（建议）
- 呈现方式：
  - 卡片右上角红/绿箭头
  - 异常模块外框高亮 + 告警图标
  - 告警列表汇总显示（最多Top 10）

## 六、数据模型建议（前后端可直接对接）

### 1）事实表（按日）
- `fact_sales_daily`：平台/店铺/类目/SKU销售指标
- `fact_service_daily`：客服接待与满意度
- `fact_warehouse_daily`：发货与库存快照
- `fact_marketing_daily`：推广花费与转化
- `fact_after_sales_daily`：退款与售后处理
- `fact_project_daily`：运营项目进度
- `fact_design_daily`：美工任务进度

### 2）维表
- `dim_date`
- `dim_platform`
- `dim_product_sku`（含上述产品清单及扩展SKU）
- `dim_team_member`
- `dim_campaign_item`

### 3）统一返回结构（示例约定）
- 查询参数：`date`, `platform[]`, `category[]`
- 返回：
  - `summary`: 总览指标
  - `modules`: 各模块数据块
  - `alerts`: 异常列表
  - `meta`: `last_update_time`, `data_version`

---

## 七、前端实现建议（工程可落地）
- 技术栈：Vue3/React + ECharts + AntV G2Plot（甘特可用X6/第三方组件）
- 轮询策略：页面每10分钟请求一次 `/api/dashboard?date=YYYY-MM-DD`
- 性能：首屏接口聚合，二级明细按需加载
- 容错：接口失败时保留最近一次成功快照，并显示“数据加载异常”提示条
- 权限：按角色控制“金额字段脱敏/显示完整值”

---

## 八、交付清单（给开发与业务）
1. 高保真页面（1920×1080）
2. 组件清单（KPI卡、环图、柱图、折线、甘特、任务表）
3. 字段字典与接口契约文档
4. 告警规则配置表（阈值可调）
5. 每日更新作业说明（ETL完成时间、失败补偿机制）

该方案可直接进入UI设计与前端开发阶段，并满足“多平台、全链路、日更新、可追溯历史”的管理决策需求。
